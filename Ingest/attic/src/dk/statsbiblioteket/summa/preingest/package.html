!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 3.2 Final//EN"> Pre-ingest is used to bring data into a intestable form.
Pre-ingest operations are chain-able in the ingest work flow (Filters).
A pre-ingest filter-chain must always terminate in a state where all meta data to be further processed is i valid XML and stored with the .xml file extension.
This can be used to format conversion, data normalization/enrichments or what ever the implementor needs.
To extend with your own Filter, simply implement the {@link dk.statsbiblioteket.summa.preingest.IngestFilter}
interface and add the filter to the configuration.
